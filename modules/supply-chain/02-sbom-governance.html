What is an SBOM?

A Software Bill of Materials (SBOM) is a formal, machine-readable inventory of all components, libraries, and dependencies that make up a software application. Think of it as a "nutrition label" for software — it tells you exactly what ingredients went into building your application.

SBOMs have become a regulatory requirement following Executive Order 14028 (Improving the Nation's Cybersecurity), which mandates SBOMs for all software sold to the U.S. federal government. Similar regulations are emerging globally, including the EU Cyber Resilience Act.

Generating SBOMs with Mend

Mend automatically generates comprehensive SBOMs for all scanned projects. SBOMs include direct and transitive dependencies, license information, and vulnerability status.

# Generate SBOM in CycloneDX format
mend sbom generate --dir ./my-project --format cyclonedx --output sbom.json

# Generate SBOM in SPDX format
mend sbom generate --dir ./my-project --format spdx --output sbom.spdx.json

# Generate SBOM with vulnerability data included
mend sbom generate --dir ./my-project --format cyclonedx --include-vulns --output sbom-with-vulns.json

# Export SBOM for a specific project from the dashboard
mend sbom export --project "my-app" --format cyclonedx

💡 SBOM Best Practices

Generate SBOMs as part of your CI/CD pipeline on every release. Store them alongside your build artifacts. This creates an auditable trail of exactly what components were included in each version of your software.

Governance & Policy Enforcement

Mend provides policy enforcement capabilities that allow organizations to define and automate governance rules across all repositories:

  • License Policies — Block or flag packages with copyleft licenses (GPL, AGPL) in proprietary products
  • Vulnerability Policies — Set severity thresholds that must be met before code can be merged or deployed
  • Age & Maintenance Policies — Flag dependencies that haven't been updated in over 12 months
  • Provenance Policies — Require packages to have verified build provenance (SLSA attestations)
  • Approval Workflows — Require security team approval for new dependencies above a risk threshold

Compliance Frameworks

Mend helps organizations meet requirements from multiple compliance frameworks:

Maintain inventory of third-party components and track known vulnerabilities.
