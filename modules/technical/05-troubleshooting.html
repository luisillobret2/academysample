Common Integration Issues

As a Technical SE, you'll encounter common integration and deployment issues. Knowing how to quickly diagnose and resolve these issues builds customer confidence and accelerates deployment.

Top 5 Integration Issues

🔒

Authentication Failures

API key expired, incorrect permissions, SSO misconfiguration, token scope too narrow.

📋

Missing Scan Results

Unsupported package manager, no manifest file found, gitignore excluding dependency files.

🔧

CI/CD Pipeline Failures

Timeout issues, exit code misinterpretation, missing environment variables, network connectivity.

📡

Webhook Delivery Failures

Firewall blocking, incorrect URL, SSL certificate issues, payload format mismatch.

Diagnostic Toolkit

Use these tools and techniques to quickly diagnose issues:

CLI Diagnostics

# Check CLI version and configuration
mend version
mend config show

# Verbose scan output for debugging
mend sca scan --verbose --debug

# Test API connectivity
mend auth verify

# Check supported file types in a directory
mend sca scan --dry-run --dir ./project

# Generate diagnostic bundle for support
mend diagnostic --output diagnostic.zip

Common Error Messages & Solutions

# Error: "No supported manifest files found"
# Solution: Check file paths, ensure package.json/pom.xml/requirements.txt
#           exists and isn't gitignored

# Error: "Authentication failed"
# Solution: Verify API key hasn't expired, check organization assignment,
#           ensure key has correct scope (org:read, project:write)

# Error: "Scan timeout after 300 seconds"
# Solution: For large monorepos, increase timeout with --timeout 600
#           Or split into sub-project scans

# Error: "Rate limit exceeded"
# Solution: Add delay between scans in CI/CD, use scan scheduling,
#           or contact Mend for enterprise rate limit increase

Performance Optimization

Large enterprise deployments may encounter performance challenges. Use these optimization strategies:

  • Incremental Scanning — Only scan changed dependencies rather than full project on every commit
  • Parallel Scanning — Configure CI/CD to run Mend scans in parallel with other pipeline stages
  • Caching — Enable scan result caching to speed up repeated scans of unchanged dependencies
  • Selective Scanning — Use path filters to exclude test dependencies and non-production code from critical scans
  • Monorepo Optimization — Split large monorepos into logical scan units with shared caching

Support Escalation Process

When you can't resolve an issue in the field, follow the escalation process:

  1. Level 1: Self-Service — Check docs.mend.io, knowledge base, and community forums
  2. Level 2: Partner Support — Contact partner support channel with diagnostic bundle attached
  3. Level 3: Engineering Escalation — For product bugs or feature gaps, escalate through your channel manager
  4. Level 4: Executive Escalation — For deal-blocking issues, engage your Mend partner executive sponsor

Creating an Effective Support Ticket

  • Include the diagnostic bundle (mend diagnostic)
  • Describe the expected vs. actual behavior
  • Include the exact error message and stack trace
  • Specify the environment (OS, CI/CD platform, SCM, language versions)
  • Note the business impact and urgency

💡 Support Tip

80% of support issues are resolved by checking three things: API key validity, network connectivity to Mend APIs, and correct manifest file paths. Always check these first before escalating.
