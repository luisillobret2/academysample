Enterprise Integration Architecture

Enterprise customers require Mend to integrate seamlessly with their existing security and development infrastructure. Understanding common integration patterns helps you design solutions that maximize value.

Integration Categories

GitHub, GitLab, Bitbucket, Azure DevOps. Automated scanning on push, PR comments, and branch policies.

📡

Communication

Slack, Microsoft Teams, PagerDuty, OpsGenie. Real-time alerts and digest notifications.

Enterprise security teams require Mend data in their security operations platforms:

Splunk Integration

# Forward Mend findings to Splunk via webhook
# Configure in Mend: Settings > Integrations > Webhooks

# Webhook payload includes:
# - Finding severity, CVE, package, version
# - Repository and branch information
# - Remediation recommendation
# - CVSS score and reachability status

# Splunk HEC (HTTP Event Collector) configuration
curl -X POST "https://splunk.company.com:8088/services/collector/event" \
    -H "Authorization: Splunk $HEC_TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
        "event": {
            "source": "mend",
            "severity": "critical",
            "cve": "CVE-2024-12345",
            "package": "lodash@4.17.20",
            "reachable": true
        }
    }'

Custom Webhook Architecture

Webhooks enable event-driven architectures that react to Mend findings in real-time:

# Example: Auto-remediation webhook handler
from flask import Flask, request
import subprocess

app = Flask(__name__)

@app.route('/mend-webhook', methods=['POST'])
def handle_finding():
    data = request.json
    event_type = data.get('eventType')

    if event_type == 'NEW_VULNERABILITY':
        severity = data['finding']['severity']
        package = data['finding']['package']
        fix_version = data['finding']['fixVersion']

        if severity == 'CRITICAL' and fix_version:
            # Auto-create remediation PR
            create_fix_pr(
                repo=data['repository'],
                package=package,
                target_version=fix_version
            )
            notify_slack(f"Auto-fix PR created for {package}")
        elif severity == 'HIGH':
            create_jira_ticket(data)

    return {'status': 'ok'}, 200

Multi-SCM Architecture

Enterprise customers often use multiple source code management platforms. Mend provides unified scanning across all of them:

  • Unified Dashboard — Single pane of glass for findings across GitHub, GitLab, Bitbucket, and Azure DevOps
  • Consistent Policies — Same security policies enforced regardless of SCM platform
  • Cross-Platform Reporting — Aggregate metrics and compliance data across all platforms
  • SCM-Native Experience — PR comments and status checks work natively in each platform

💡 Integration Tip

Start integrations with the highest-value, lowest-effort connections first: SCM integration (immediate scanning value) and Slack notifications (immediate visibility). Then layer on JIRA, SIEM, and custom webhooks as the deployment matures.
