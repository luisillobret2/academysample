Demo Mastery

🕒 45 min 📚 4 sections ⭐ 175 XP Level 2 - Intermediate

🎯 Learning Objectives

  • Prepare a reliable, reset-able demo environment
  • Deliver the standard Mend.io demo flow for SCA and SAST
  • Tell an exploitability-to-fix story instead of a feature tour
  • Handle live questions and recover gracefully from failures

Prepare the Environment

A great demo is 80% preparation. Set up a known-good tenant with realistic data before you ever share your screen.

  1. Use a dedicated demo org with a seeded vulnerable project (e.g., a sample Node/Java app).
  2. Pre-run a scan so findings load instantly; never scan live unless it’s the point.
  3. Prepare a reset script so the tenant returns to a clean state between demos.
  4. Have a backup recording in case connectivity fails.

The Standard Demo Flow

🔍

1. Show the Noise

Open raw findings, then apply reachability to collapse the list to what’s exploitable.

🎯

2. Prioritize

Sort by EPSS and reachability so the customer sees a short, defensible fix list.

🔄

3. Auto-Remediate

Trigger a fix PR and show the diff, the highlight that lands with engineering.

📊

4. Report

Close on an executive dashboard and SBOM export for the compliance buyer.

Storytelling & Recovery

Narrate outcomes, not clicks. Anchor each step to the pain you heard in discovery.

💡 Tell, Don’t Tour

“Remember the 400 open findings your team can’t triage? Watch what reachability does to that number.” Tie every screen back to their words.

⚠️ When Something Breaks

Stay calm, narrate what should happen, and switch to your backup recording. Never debug live in front of the buyer.
