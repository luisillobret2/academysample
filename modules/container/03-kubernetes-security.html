Kubernetes Security Challenges

Kubernetes orchestrates containerized workloads at scale, but introduces a complex security surface. Misconfigurations are the leading cause of Kubernetes security incidents — from overly permissive RBAC roles to exposed dashboards and secrets stored in plaintext.

The 4C's of Cloud-Native Security

☁️

Cloud

Cloud provider infrastructure: network security, IAM, encryption at rest, API endpoint protection.

⎈

Cluster

Kubernetes cluster: API server security, etcd encryption, admission controllers, audit logging.

📦

Container

Container runtime: image scanning, least-privilege users, read-only filesystems, resource limits.

💻

Code

Application code: dependency scanning (SCA/SAST), secrets management, input validation.

Kubernetes Workload Scanning

Mend scans Kubernetes manifests, Helm charts, and Kustomize configurations to identify security misconfigurations and policy violations:

Common Misconfigurations Detected

# INSECURE: Running as root with elevated privileges
apiVersion: v1
kind: Pod
spec:
  containers:
  - name: app
    image: myapp:latest          # No image tag pinning
    securityContext:
      runAsUser: 0               # Running as root
      privileged: true           # Full host access
      allowPrivilegeEscalation: true
    resources: {}                # No resource limits

# SECURE: Hardened configuration
apiVersion: v1
kind: Pod
spec:
  containers:
  - name: app
    image: myapp:v2.1.0@sha256:abc123...  # Pinned with digest
    securityContext:
      runAsNonRoot: true
      runAsUser: 1000
      readOnlyRootFilesystem: true
      allowPrivilegeEscalation: false
      capabilities:
        drop: ["ALL"]
    resources:
      limits:
        cpu: "500m"
        memory: "256Mi"
      requests:
        cpu: "100m"
        memory: "128Mi"

Scanning Kubernetes Resources

# Scan Kubernetes manifests
mend k8s scan --dir ./k8s/

# Scan Helm charts
mend k8s scan --helm ./charts/myapp/

# Scan a running cluster
mend k8s scan --cluster my-cluster --context production

# Scan with CIS Kubernetes Benchmark
mend k8s scan --dir ./k8s/ --benchmark cis-1.8

Admission Control & Runtime Protection

Mend provides admission controllers that prevent insecure workloads from being deployed to your clusters:

  • Image Policy — Only allow images from trusted registries that have passed vulnerability scanning
  • Configuration Policy — Enforce security contexts, resource limits, and network policies on all deployments
  • Runtime Monitoring — Continuously scan running workloads for newly discovered vulnerabilities and alert when action is needed
  • Namespace Isolation — Enforce network policies that restrict cross-namespace communication

💡 Shift Left for Kubernetes

Scan Kubernetes manifests in your CI/CD pipeline, not just at deployment time. Catching misconfigurations in a pull request is far cheaper than detecting them in production.
