GitOps & SCM Integration

🕒 40 min 📚 4 sections ⭐ 175 XP Level 3 - Expert

🎯 Learning Objectives

  • Connect Mend.io to major source control platforms
  • Enable PR checks and automated fix pull requests
  • Apply policy as code stored alongside the repository
  • Scale onboarding across many repositories at once

Connect Your SCM

Mend.io integrates natively with GitHub, GitLab, Bitbucket, and Azure DevOps. Repository integrations scan automatically on every commit and pull request.

  1. Install the Mend app or configure an SCM token with least-privilege scopes.
  2. Select the organizations and repositories to onboard.
  3. Enable PR status checks and automated fix PRs.
  4. Confirm the first scan and review inline PR annotations.

Policy as Code

Store security policy in the repo so it is versioned, reviewed, and consistent across teams.

# .mend/policy.yml
sca:
  fail_on: high
  reachable_only: true
license:
  deny: [GPL-3.0, AGPL-3.0]
remediation:
  auto_pr: true

💡 Review Policy Like Code

Because policy lives in Git, changes go through pull requests, giving security and engineering shared ownership.

Scale Onboarding

Use organization-level integration and templates to onboard hundreds of repos without per-repo toil.

⚠️ Mind Token Scope

Grant the minimum permissions required. Over-scoped SCM tokens are a supply-chain risk in themselves.
