Pipeline Security Fundamentals

🕒 45 min 📚 4 sections ⭐ 175 XP Level 3 - Expert

🎯 Learning Objectives

  • Place security scans at the right stages of a CI/CD pipeline
  • Define fail-fast gates without blocking developer velocity
  • Use SARIF output to integrate results with native tooling
  • Balance blocking policy with informational reporting

Where Security Fits in the Pipeline

Effective pipelines scan early and often. Mend.io runs at commit, PR, and build stages so issues surface where they are cheapest to fix.

💻

Commit

IDE and pre-commit scans catch issues before code is pushed.

🔄

Pull Request

PR checks post findings and open fix PRs automatically.

🏗

Build

Full SCA/SAST scans gate the pipeline on policy violations.

📊

Release

SBOM generation and attestation for every artifact.

Fail-Fast Gates

Use severity thresholds to fail builds on real risk while letting low-priority findings pass as informational.

# GitHub Actions example
- name: Mend SCA scan
  run: mend sca scan --dir . --fail-on high --format sarif > mend.sarif

- name: Upload results
  uses: github/codeql-action/upload-sarif@v3
  with:
    sarif_file: mend.sarif

💡 Gate on Reachable + High

Blocking only on reachable, high-severity findings keeps signal high and developer frustration low.

Report Without Blocking

Not every finding should stop a build. Publish informational results to dashboards and SARIF so teams can triage on their own cadence.

⚠️ Avoid Alert Fatigue

Blocking on everything trains developers to bypass the pipeline. Start informational, then tighten gates as noise drops.
