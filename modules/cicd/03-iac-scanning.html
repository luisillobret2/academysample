IaC Scanning

🕒 40 min 📚 4 sections ⭐ 175 XP Level 3 - Expert

🎯 Learning Objectives

  • Explain why infrastructure as code needs security scanning
  • Detect misconfigurations in Terraform, Kubernetes, and CloudFormation
  • Enforce IaC policy in the pipeline before deployment
  • Remediate common cloud misconfigurations

Why Scan IaC?

Misconfigured infrastructure is a leading cause of cloud breaches. Scanning IaC catches issues like public buckets and open security groups before they are ever provisioned.

MisconfigurationRisk
Public S3 bucketData exposure
0.0.0.0/0 security groupOpen attack surface
Unencrypted volumesData-at-rest exposure
Privileged containersContainer escape

Scan Before Deploy

Run IaC scans in the pipeline so misconfigurations fail the build, not production.

# Scan IaC files in the pipeline
mend iac scan --dir ./infra --fail-on high

# Common targets
#   Terraform (*.tf), Kubernetes manifests (*.yaml),
#   CloudFormation (*.json / *.yaml), Helm charts

💡 Shift Cloud Security Left

Fixing a public bucket in a Terraform PR takes seconds; fixing it after a breach takes far longer and carries real cost.

Remediate & Enforce

Pair detection with guardrails: policy blocks high-risk changes and suggested fixes speed remediation.

⚠️ Watch Module Sources

Third-party Terraform and Helm modules can introduce insecure defaults. Scan them like any other dependency.
