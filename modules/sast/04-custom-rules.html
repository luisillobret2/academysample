Organization-Specific Security Standards

Every organization has unique security requirements beyond standard vulnerability detection. Mend SAST allows security teams to define custom rules and policies that enforce organization-specific coding standards, regulatory requirements, and security best practices.

Automation Workflows in the Mend AppSec Platform

The Mend AppSec Platform provides Automation Workflows that allow you to automatically enforce security rules and streamline processes across your organization:

🛠

Severity Thresholds

Set minimum severity levels that trigger different actions — auto-create Jira tickets for Critical, send Slack alerts for High, log Medium for quarterly review.

🔒

Compliance Rules

Map SAST findings to compliance frameworks (OWASP Top 10, SANS Top 25, PCI DSS) and generate compliance-specific reports for auditors.

👥

Team Routing

Automatically assign findings to the right team based on file path, severity, or finding type. Security-critical findings go to the security team; code quality issues go to developers.

📢

Notification Channels

Configure alerts via email, Slack, Microsoft Teams, or webhooks. Set different notification rules for different severity levels and teams.

Managing Access & Organizational Structure

The Mend AppSec Platform uses a hierarchical structure to organize security management across large enterprises:

  • Organization — Top-level entity representing your company. Organization-wide policies apply to all applications and projects.
  • Applications — Logical groupings (e.g., by product, team, or business unit). Each application can have its own policies and access controls.
  • Projects — Individual repositories or codebases. Scan results and findings are tracked at the project level.

Access Levels

Mend supports role-based access control with granular permissions at each level of the hierarchy, ensuring security teams have admin access while developers see only their relevant projects.
