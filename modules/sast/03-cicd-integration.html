CI/CD Pipeline Integration

🕒 40 min 📚 4 sections ⭐ 200 XP Level 2 - Intermediate

All Within Your Repo

Mend SAST integrates directly into your existing CI/CD pipelines, providing security feedback on every pull request without requiring developers to leave their workflow. All checks run within your repository infrastructure.

Supported Platforms

💫

GitHub Actions

Native GitHub Action for SAST scanning on every PR. Results appear as check annotations directly on changed lines.

💫

GitLab CI

GitLab CI/CD integration with SARIF report upload and merge request annotations for vulnerability findings.

💫

Azure DevOps

Azure Pipelines task for SAST scanning with results integrated into Azure DevOps pull request experience.

💫

Jenkins / Bitbucket

CLI-based integration for Jenkins pipelines and Bitbucket Pipelines using the Mend CLI tool.

GitHub Actions Example

Here’s a typical GitHub Actions workflow that runs Mend SAST on every pull request:

name: Mend SAST Scan
on:
  pull_request:
    branches: [main, develop]

jobs:
  sast-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run Mend SAST
        uses: mend-toolkit/mend-sast-action@v1
        with:
          api-key: ${{ secrets.MEND_API_KEY }}
          fail-on-severity: high
          output-format: sarif

      - name: Upload SARIF Results
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: mend-sast-results.sarif

💡 Security Gates

The fail-on-severity parameter creates a security gate — if any finding at or above the specified severity is detected, the CI check fails and the PR cannot be merged. This prevents new vulnerabilities from entering the codebase while allowing teams to set the threshold appropriate for their risk tolerance.

Repository Integrations

Beyond CI/CD pipelines, Mend offers native repository integrations that scan automatically on every commit and PR:

Integration Trigger Results Location
Mend Repository Integration Every commit and PR automatically PR comments, checks, Mend dashboard
Mend CLI in Pipeline CI/CD workflow step CLI output, SARIF upload, dashboard
IDE Extensions Real-time as you code Inline annotations in the editor
Scheduled Scans Cron schedule (daily/weekly) Mend dashboard, email notifications
