Mend Renovate Setup & Configuration

🕐 40 min 📚 4 sections ⭐ 200 XP Level 2 - Intermediate

What is Mend Renovate?

Mend Renovate is the industry-leading open-source tool for automated dependency updates. It creates pull requests to update your dependencies, reducing security risk and technical debt with minimal manual effort. Used by thousands of organizations including Google, Mozilla, and Uber.

Why Renovate?

🔄

Automated Updates

Automatically detects outdated dependencies and creates PRs with changelogs and release notes.

📦

Multi-Ecosystem

Supports 90+ package managers: npm, pip, Maven, NuGet, Go, Cargo, Helm, Docker, Terraform, and more.

🔧

Highly Configurable

Fine-grained control over update scheduling, grouping, auto-merge rules, and version constraints.

🔐

Security-First

Prioritizes security updates and can be configured to fast-track vulnerability fixes.

Getting Started with Renovate

GitHub App Installation

# 1. Install the Mend Renovate GitHub App
#    Go to: github.com/apps/renovate
#    Click "Install" and select your repositories

# 2. Renovate will create an onboarding PR with a default config
#    Review and merge the PR to activate

# 3. Default renovate.json
{
  "$schema": "https://docs.renovatebot.com/renovate-schema.json",
  "extends": [
    "config:recommended"
  ]
}

Self-Hosted Renovate

# Run Renovate as a Docker container
docker run --rm \
  -e RENOVATE_TOKEN=YOUR_GITHUB_TOKEN \
  -e RENOVATE_REPOSITORIES='["org/repo1","org/repo2"]' \
  renovate/renovate

# Or as a GitHub Action
name: Renovate
on:
  schedule:
    - cron: '0 4 * * *'
jobs:
  renovate:
    runs-on: ubuntu-latest
    steps:
      - uses: renovatebot/github-action@v40
        with:
          token: ${{ secrets.RENOVATE_TOKEN }}
          configurationFile: renovate.json

Advanced Configuration

// renovate.json - Advanced configuration
{
  "extends": ["config:recommended"],
  "schedule": ["before 7am on Monday"],
  "timezone": "America/New_York",
  "packageRules": [
    {
      "description": "Auto-merge patch updates for non-major deps",
      "matchUpdateTypes": ["patch"],
      "automerge": true,
      "automergeType": "pr"
    },
    {
      "description": "Group all ESLint packages",
      "matchPackagePatterns": ["^eslint"],
      "groupName": "ESLint"
    },
    {
      "description": "Prioritize security updates",
      "matchUpdateTypes": ["patch", "minor"],
      "matchCategories": ["security"],
      "schedule": ["at any time"],
      "automerge": true,
      "prPriority": 10
    },
    {
      "description": "Pin Docker digests",
      "matchDatasources": ["docker"],
      "pinDigests": true
    }
  ],
  "vulnerabilityAlerts": {
    "enabled": true,
    "labels": ["security"]
  }
}

Renovate Best Practices

  • Start with Recommended Config — The config:recommended preset handles most common scenarios well
  • Enable Auto-merge for Patches — When CI passes, auto-merge patch updates to reduce PR noise
  • Group Related Updates — Group monorepo packages (e.g., all @angular/* packages) into single PRs
  • Schedule Updates — Run updates on a schedule that aligns with your team's review capacity
  • Fast-Track Security — Configure security updates to bypass scheduling and get merged ASAP
  • Pin Docker Digests — Pin container images to digests for reproducible builds

💡 Pro Tip

Use Renovate's Dependency Dashboard (an issue in your repo) to get an overview of all pending updates and manually trigger or dismiss specific updates.
