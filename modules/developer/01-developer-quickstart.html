Developer Quickstart with Mend

🕒 30 min 📚 4 sections ⭐ 150 XP Level 1 - Beginner

Getting Started in 5 Minutes

This quickstart guide gets developers scanning their first project with Mend in under 5 minutes. No security expertise required — Mend integrates directly into your existing development workflow.

Step 1: Install the Mend CLI

# Install via npm (Node.js)
npm install -g @mend/cli

# Or via Homebrew (macOS)
brew install mend-io/tap/mend-cli

# Verify installation
mend --version

Step 2: Authenticate

# Login with your Mend account
mend auth login

# Or set your API key as an environment variable
export MEND_API_KEY="your-api-key-here"

Step 3: Scan Your Project

# Run a full scan (SCA + SAST)
mend scan --dir .

# Run SCA only
mend sca scan --dir .

# Run SAST only
mend sast scan --dir .

Understanding Your Results

After scanning, Mend categorizes findings to help you focus on what matters:

🔴

Critical & High

Vulnerabilities that are exploitable and have available fixes. Address these immediately — especially those marked as Reachable.

🟠

Medium

Vulnerabilities with moderate risk. Schedule remediation in your next sprint, prioritizing those with high EPSS scores.

🟢

Low & Info

Minor issues or informational findings. Address during regular dependency updates or as part of tech debt sprints.

🔓

License Issues

Dependencies with licenses that may conflict with your project’s licensing requirements. Review with your legal team.
