IDE Plugins & Developer Workflow

🕐 35 min 📚 4 sections ⭐ 175 XP Level 1 - Beginner

Shift-Left with IDE Integration

The fastest way to fix vulnerabilities is to find them before they leave the developer's machine. Mend's IDE plugins bring security scanning directly into VS Code, IntelliJ, and other popular editors, showing vulnerabilities inline as developers write code.

Supported IDEs

💻

VS Code Extension

Real-time SCA and SAST scanning with inline vulnerability annotations, quick-fix suggestions, and dependency tree visualization.

💻

IntelliJ Plugin

Full integration with IntelliJ IDEA, WebStorm, and PyCharm. Scans on save with results in the Problems panel.

💻

Eclipse Plugin

Enterprise Java development support with Maven/Gradle dependency scanning and fix recommendations.

💻

Visual Studio

.NET and NuGet package scanning with inline annotations in Visual Studio 2019/2022.

Installing the VS Code Extension

# Install from the VS Code Marketplace
# 1. Open VS Code
# 2. Go to Extensions (Ctrl+Shift+X)
# 3. Search for "Mend"
# 4. Click Install

# Or install from the command line:
code --install-extension mend.mend-vscode

# Configure your API key
# Settings > Extensions > Mend > API Key
# Enter your organization's API key

Key Features

  • Inline Annotations — Vulnerable dependencies are highlighted with severity indicators directly in package files
  • Quick Fixes — One-click upgrade to the nearest safe version
  • Dependency Tree — Visualize your full dependency tree with vulnerability indicators at every level
  • SAST Findings — Code-level security issues highlighted inline with remediation guidance

Developer Workflow Integration

Mend fits into existing development workflows without disrupting developer productivity:

  • Pre-commit — Scan before committing to catch issues earliest
  • On Save — IDE plugin scans automatically when files are saved
  • PR Check — Automated scanning on every pull request with inline comments
  • CLI — Run scans from the terminal for scripting and automation

💡 Developer Tip

Configure the IDE plugin to show only high and critical vulnerabilities initially. As your team builds the habit of addressing security issues, gradually include medium-severity findings.

PR Integration & Review

Mend automatically comments on pull requests with security findings, making security review a natural part of code review:

  • New Vulnerability Alerts — Only flags issues introduced in the PR, not pre-existing ones
  • Fix Suggestions — Suggests specific version upgrades or code changes to resolve the issue
  • Policy Checks — Pass/fail status based on your organization's security policies
  • License Alerts — Flags new dependencies with incompatible licenses
