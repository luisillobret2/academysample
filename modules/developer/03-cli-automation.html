The Mend CLI

The Mend CLI is a powerful command-line tool for running security scans, managing policies, and integrating Mend into any automation workflow. It supports SCA, SAST, secrets detection, and container scanning from a single binary.

Installation

# macOS / Linux
curl -fsSL https://downloads.mend.io/cli/install.sh | sh

# Or with Homebrew
brew install mend-io/tap/mend-cli

# Windows
# Download from https://downloads.mend.io/cli/latest/mend-cli-win.exe

# Verify installation
mend version

# Authenticate
mend auth login --api-key YOUR_API_KEY

Common CLI Commands

SCA Scanning

# Scan current directory for open-source vulnerabilities
mend sca scan

# Scan with specific output format
mend sca scan --format json --output results.json

# Scan and fail on critical/high
mend sca scan --fail-on high

# Generate SBOM
mend sca sbom --format cyclonedx --output sbom.json

SAST Scanning

# Run static analysis
mend sast scan --dir ./src

# Scan specific languages
mend sast scan --dir ./src --languages java,python

# Generate SARIF report
mend sast scan --dir ./src --format sarif --output sast-results.sarif

Container Scanning

# Scan a local container image
mend container scan myapp:latest

# Scan from a registry
mend container scan registry.example.com/myapp:v1.2.3

# Scan a Dockerfile
mend container scan --dockerfile ./Dockerfile

CI/CD Pipeline Integration

The CLI integrates seamlessly into any CI/CD pipeline:

GitHub Actions

name: Security Scan
on: [push, pull_request]
jobs:
  mend-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install Mend CLI
        run: curl -fsSL https://downloads.mend.io/cli/install.sh | sh
      - name: Run SCA Scan
        run: mend sca scan --fail-on high
        env:
          MEND_API_KEY: ${{ secrets.MEND_API_KEY }}
      - name: Run SAST Scan
        run: mend sast scan --dir ./src --fail-on critical

GitLab CI

mend-security-scan:
  stage: test
  script:
    - curl -fsSL https://downloads.mend.io/cli/install.sh | sh
    - mend sca scan --format json --output gl-dependency-scanning-report.json
    - mend sast scan --dir ./src --format json --output gl-sast-report.json
  artifacts:
    reports:
      dependency_scanning: gl-dependency-scanning-report.json
      sast: gl-sast-report.json

Automation & Scripting

The CLI's JSON output mode enables powerful automation workflows:

# Custom alerting script
#!/bin/bash
RESULTS=$(mend sca scan --format json)
CRITICAL_COUNT=$(echo "$RESULTS" | jq '[.vulnerabilities[] | select(.severity == "critical")] | length')

if [ "$CRITICAL_COUNT" -gt 0 ]; then
    echo "ALERT: $CRITICAL_COUNT critical vulnerabilities found!"
    # Send to Slack
    curl -X POST "$SLACK_WEBHOOK" \
        -H 'Content-type: application/json' \
        -d "{"text": "Security Alert: $CRITICAL_COUNT critical vulnerabilities found in $(basename $PWD)"}"
fi

💡 Automation Tip

Use the CLI's --format json output with tools like jq to build custom dashboards, alerts, and compliance reports tailored to your organization's needs.
