Zero In on True Risks Without the Noise

Traditional SCA tools report every CVE found in your dependency tree, regardless of whether the vulnerability can actually be triggered. This creates massive alert fatigue — teams waste time investigating findings that pose no real threat.

Mend SCA’s Reachability Analysis solves this by determining whether your code actually calls the vulnerable function in a dependency. It analyzes the call graph between your application code and the vulnerable functions in both direct and transitive dependencies.

💡 The Impact of Reachability

Studies show that only 10-20% of reported CVEs are actually reachable in any given application. By filtering out unreachable vulnerabilities, Mend helps teams focus remediation efforts on the vulnerabilities that can actually be exploited, reducing noise by up to 80%.

How Reachability Analysis Works

Mend’s reachability engine performs a multi-step analysis:

  1. Dependency Resolution — Builds the complete dependency tree from manifest and lockfiles, identifying all direct and transitive dependencies
  2. Call Graph Construction — Maps all function calls from your application code into dependency code, following the execution path through multiple layers
  3. Vulnerability Mapping — Cross-references the call graph with known vulnerable functions for each CVE in the dependency
  4. Reachability Determination — Classifies each vulnerability as Reachable, Unreachable, or Potentially Reachable based on whether a call path exists

Reachability Statuses

Status Meaning Action
Reachable Your code calls the vulnerable function — this vulnerability can be exploited Prioritize immediate remediation
Potentially Reachable Analysis is inconclusive — the call path may exist under certain conditions Investigate and schedule remediation
Unreachable Your code does not call the vulnerable function — exploitation requires code changes Deprioritize; remediate in normal cycle

Reachability in Practice

Consider a real-world scenario: your application uses the xstream library for XML serialization. A Critical CVE (CVE-2020-1245) is published affecting xstream's deserialization handler.

⚠️

Without Reachability

Traditional SCA reports this as Critical. Your security team drops everything to investigate, pulling developers off feature work. After hours of analysis, they discover your code never uses xstream's deserialization — wasted effort.

✓

With Mend Reachability

Mend instantly identifies the CVE as Unreachable — your code only uses xstream for serialization, never the vulnerable deserialization path. The team schedules the update for the next sprint instead of treating it as an emergency.
