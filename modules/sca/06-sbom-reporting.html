SBOM Formats & Standards

Mend supports two industry-standard SBOM formats:

📜

SPDX 2.2

Linux Foundation standard (ISO/IEC 5962:2021). The de facto standard for government compliance, including US Executive Order 14028. Focuses on license and copyright information alongside component inventory.

📄

CycloneDX 1.4

OWASP standard designed specifically for security use cases. Rich vulnerability information including VEX (Vulnerability Exploitability eXchange) data for communicating whether a product is affected by a specific CVE.

Generating an SBOM

Generate an SBOM through the Mend platform or CLI:

# Generate an SPDX SBOM
mend sca report --type sbom --format spdx --output sbom-spdx.json

# Generate a CycloneDX SBOM
mend sca report --type sbom --format cyclonedx --output sbom-cdx.json

# Generate a Dependencies SBOM via the Mend AppSec Platform
# Navigate to: Reports > Create Report > Dependencies SBOM
# Select: SBOM Standard (SPDX 2.2 or CycloneDX 1.4)
# Select: Format (JSON or XML)
# Select: Scope (Organization, Application, or Project)

Mend Dashboards & Reporting

The Mend AppSec Platform provides comprehensive dashboards for monitoring your security posture across the entire organization:

📊

Organization Dashboard

Executive view of security posture across all applications: total findings, severity trends, remediation velocity, and compliance metrics.

💻

Application Dashboard

Per-application view showing dependency inventory, vulnerability breakdown, license compliance status, and historical trend data.

📋

Compliance Reports

Generate audit-ready reports for regulatory compliance (SOC 2, ISO 27001, FedRAMP) with evidence of vulnerability management practices.

🔗

API Access

The Mend API 3.0 provides programmatic access to all scan results, vulnerability data, and reporting capabilities for integration with custom dashboards and workflows.
