What is Mend SCA?

Mend SCA (Software Composition Analysis) gives organizations full visibility and control over open-source usage and security. It makes it easy for developers to remediate open-source risk directly from the tools they already use. Mend SCA can issue real-time alerts with automatic remediation capabilities, and can even proactively block malicious packages and licensing violations.

How SCA Scanning Works

Mend SCA scans your project’s dependency manifests (e.g., package.json, pom.xml, requirements.txt, go.mod) and lockfiles to build a complete dependency tree. It then cross-references every component against:

  • Mend Vulnerability Database — Curated from NVD, GitHub Advisories, and proprietary research
  • Malicious Package Database — Real-time feeds identifying typosquatting, dependency confusion, and compromised packages
  • License Database — Complete license classification for compliance enforcement

Getting Started with Mend AppSec Platform

Before running your first SCA scan, you need to set up your environment within the Mend AppSec Platform:

  1. Set Up Sign-In (SSO) — Configure secure login access for your organization with seamless SSO integration via SAML 2.0 or OpenID Connect
  2. Configure Your Organization — Set up your account structure with organizations, applications, and projects that mirror your development teams
  3. Connect Repository Integrations — Link your source code repositories from GitHub, GitLab, Bitbucket, or Azure DevOps for automatic scanning
  4. Configure Automation Workflows — Set up security policies that automatically enforce rules and streamline remediation processes
  5. Run Your First Scan — Use the Mend CLI or repository integration to scan a project and review findings in the dashboard

💡 Scanning Methods

Mend SCA offers multiple scanning methods: Repository Integrations scan automatically on every commit/PR, the Mend CLI provides on-demand scanning from any terminal or CI/CD pipeline, and the Unified Agent offers advanced scanning within the CLI for complex projects.

Scan Using the Mend CLI

The Mend CLI is the primary tool for running SCA scans. It supports scanning from developer workstations, CI/CD pipelines, and automated workflows.

# Install the Mend CLI
npm install -g @mend/cli

# Authenticate with your organization
mend auth login

# Run an SCA scan on your project
mend sca scan --dir ./my-project

# Run with specific configuration
mend sca scan --dir ./my-project --scope "dependencies" --format json

Configuration Parameters

The Mend CLI supports extensive configuration through command-line flags and configuration files:

Parameter Description Default
--dir Project directory to scan Current directory
--scope Scan scope: dependencies, devDependencies, or all dependencies
--severity Minimum severity to report: low, medium, high, critical low
--format Output format: table, json, sarif table
--fail-on Fail the scan if findings exceed severity threshold none

Understanding Scan Results

After a scan completes, Mend SCA categorizes findings by severity and provides actionable context for each vulnerability:

🔎

Security Findings

View CVEs identified in your dependencies with CVSS scores, EPSS predictions, reachability status, and available fix versions.

📋

License Violations

Identify dependencies with licenses that violate your organization’s policy (e.g., GPL in a proprietary product).

⚠️

Malicious Packages

Real-time alerts for packages flagged as typosquatting, dependency confusion, or containing known malware.

📈

Dashboards

Organization-wide visibility into vulnerability trends, remediation progress, and compliance status across all projects.
