Security Policy Configuration

Mend SCA policies translate organizational risk appetite into automated decisions. A security policy defines which vulnerabilities are unacceptable based on severity, CVSS score, exploit maturity, reachability, and whether a fix is available. Policies can be scoped to the entire organization, a specific application, or a single repository.

Effective policies balance risk reduction with developer productivity. A blanket "block all CVEs" rule creates friction and alert fatigue, while a nuanced rule such as "block reachable critical CVEs with available fixes" focuses enforcement on issues that actually matter. Mend lets you combine multiple criteria so policies reflect real-world exploitability rather than raw score alone.

Policy Scope

Policies are inherited hierarchically: organization-level policies apply to every application and project, while application-level policies can add exceptions or stricter requirements for sensitive systems. Project-level policies give teams flexibility for experimental code without weakening enterprise standards.

Example Severity Threshold Policy

Policy: Block Reachable Critical/High Vulnerabilities
- Severity: Critical or High
- Reachability: Reachable only
- Fix availability: Any (including upgrade required)
- Action: Block pull request and fail build
- Exemptions: Requires security team approval via policy violation workflow

License Policy Rules

Open-source license compliance is as critical as vulnerability management. Mend SCA license policies classify licenses by risk level and define the actions to take when a forbidden or restricted license appears in your dependency tree. Categories include Permissive, Weak Copyleft, Strong Copyleft, Commercial, and Custom License Groups.

License policies evaluate the full dependency tree, including transitive dependencies that developers may not know about. A popular permissive library might pull in a strong copyleft dependency, so Mend scans the entire resolved dependency graph rather than only top-level declarations. This prevents license contamination from slipping into production.

Enforcement Actions & Policy as Code

Enforcement is where policy meets the development workflow. Mend SCA can trigger actions at multiple points: during CI/CD scans, in pull request checks, and on scheduled baseline scans. The most common enforcement actions are blocking pull requests, failing builds, creating Jira tickets, and sending notifications to Slack or Microsoft Teams.

Policy as code lets you store policy definitions in version-controlled files alongside your application code. This approach supports reproducible governance, peer review of policy changes, and integration with existing GitOps workflows. Mend supports YAML-based policy definitions that can be committed to a repository or managed centrally via the Mend API.

🛑

Block Pull Requests

Prevent merging code that violates security or license policies. The PR check surfaces the violating library, severity, and recommended fix directly in the developer workflow.

🚫

Fail Builds

CI/CD integration returns a non-zero exit code when violations are detected, ensuring that non-compliant artifacts cannot be built or deployed.

📋

Create Tickets

Automated Jira, Azure DevOps, or ServiceNow tickets route violations to the right team with full context, severity, and remediation guidance.

💬

Notify Teams

Slack, Microsoft Teams, and email notifications keep security champions and developers informed without overwhelming them.

Governance Reporting & Compliance

Governance reporting turns scan data into evidence for auditors, executives, and compliance teams. Mend provides dashboards, scheduled reports, and exportable artifacts that demonstrate policy coverage, violation trends, mean-time-to-remediate, and open-source inventory health. These reports are essential for SOC 2, ISO 27001, FedRAMP, and customer security questionnaires.

Audit trails record every policy change, exemption approval, and enforcement action. Knowing who changed a policy, when, and why is critical for compliance and forensic investigation. Mend preserves this history in the platform and exposes it through the API for integration with centralized logging and SIEM systems.

💡 Governance Tip

Schedule a weekly executive summary that shows policy violation trends and mean-time-to-remediate. Pair it with a monthly audit report of policy changes and exemptions. This combination gives leadership visibility and gives auditors a clear trail of governance activity.
