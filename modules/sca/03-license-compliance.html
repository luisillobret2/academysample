Govern Compliance of Organizational Standards

Open-source licenses dictate how software can be used, modified, and distributed. Using a library with an incompatible license in a proprietary product can expose an organization to significant legal and financial risk. Mend SCA gives your legal team the visibility and control needed to ensure open-source components meet organizational standards.

License Categories

Category License Examples Risk Level Key Restriction
Permissive MIT, Apache 2.0, BSD 2/3-Clause Low Minimal — attribution required
Weak Copyleft LGPL 2.1, LGPL 3.0, MPL 2.0 Medium Modifications to the library must be open-sourced
Strong Copyleft GPL 2.0, GPL 3.0, AGPL 3.0 High Entire derivative work must be open-sourced
Commercial / Proprietary Vendor-specific licenses Variable Requires paid license or specific terms
No License / Unknown Unlicensed code Critical All rights reserved by default — cannot be used

Mend License Detection & Enforcement

When Mend SCA detects license types that violate company policy, it issues real-time alerts with automatic remediation capabilities and can even block license violations before they become part of your code base.

How It Works

  • Automatic Detection — Mend identifies the license for every direct and transitive dependency, including dual-licensed packages
  • Copyright Tracking — Tracks copyright information including organization, author, start/end year, and additional metadata
  • Policy Rules — Define rules like “Block GPL-licensed packages in the mobile app project” or “Alert on AGPL in any project”
  • PR Blocking — Automatically fail CI/CD checks when a PR introduces a dependency with a banned license
  • Exception Management — Grant exceptions for specific libraries when legal review approves usage

💡 Common Customer Scenario

A financial services company distributes a proprietary trading platform. A developer adds a seemingly harmless open-source utility that is GPL-3.0 licensed. Without Mend, the company could be forced to open-source their entire application. With Mend’s policy enforcement, the PR is blocked with an explanation, and the developer is guided to an MIT-licensed alternative.

Setting Up License Policies

Configure license policies in the Mend AppSec Platform through Automation Workflows:

  1. Navigate to Automation Workflows — Access the policy configuration from the Mend AppSec Platform settings
  2. Create a License Policy — Define which license types are approved, restricted, or banned for each project or organization
  3. Set Enforcement Actions — Choose between Alert (notify only), Warn (non-blocking check), or Block (fail the build/PR)
  4. Configure Exceptions — Add specific package exceptions for pre-approved libraries with documentation links
  5. Enable Reporting — Generate license compliance reports for legal review and audit trail
