API & Automation at Scale

🕐 40 min 📚 4 sections ⭐ 250 XP Level 3 - Expert

Mend API Overview

The Mend API enables programmatic access to all platform capabilities — from triggering scans to exporting reports. At enterprise scale, the API is the primary interface for integrating Mend into custom workflows, building dashboards, and automating security operations.

▶

Mend API Architecture & Best Practices

7:30

API Authentication

# Generate an API key from the Mend dashboard
# Settings → API Keys → Generate New Key

# Authenticate with the API
curl -X POST https://api.mend.io/v2/auth/token \
  -H "Content-Type: application/json" \
  -d '{
    "email": "service-account@example.com",
    "apiKey": "your-api-key-here",
    "orgUuid": "org-uuid-here"
  }'

# Use the returned JWT token for subsequent requests
export MEND_TOKEN="eyJhbGciOiJSUzI1..."

curl -X GET https://api.mend.io/v2/orgs/{orgUuid}/projects \
  -H "Authorization: Bearer $MEND_TOKEN"

⚠️ Security Best Practice

Use dedicated service accounts for API access, not individual user credentials. Rotate API keys every 90 days. Store keys in a secrets manager, never in code or CI/CD configuration files.

Common API Use Cases

📊

Custom Dashboards

Pull vulnerability data into Grafana, Power BI, or custom portals for executive reporting.

⚙️

CI/CD Integration

Trigger scans, retrieve results, and enforce policies in custom build pipelines.

📄

Compliance Reporting

Auto-generate SBOM exports, license reports, and vulnerability summaries for auditors.

🔔

Alert Routing

Forward security findings to Jira, ServiceNow, PagerDuty, or custom ticketing systems.

Example: Exporting Vulnerability Report

# Get all vulnerabilities for a project
curl -X GET "https://api.mend.io/v2/orgs/{orgUuid}/projects/{projectUuid}/vulnerabilities" \
  -H "Authorization: Bearer $MEND_TOKEN" \
  -H "Accept: application/json" | jq '.vulnerabilities[] | {
    cve: .name,
    severity: .severity,
    library: .library.name,
    version: .library.version,
    fixVersion: .topFix.fixResolution,
    reachable: .reachability.isReachable
  }'

# Export SBOM for a project
curl -X GET "https://api.mend.io/v2/orgs/{orgUuid}/projects/{projectUuid}/sbom?format=cyclonedx" \
  -H "Authorization: Bearer $MEND_TOKEN" \
  -o sbom.json

Webhooks & Event-Driven Automation

Mend can push events to your systems via webhooks, enabling real-time automation:

Webhook Events

  • NEW_VULNERABILITY — Triggered when a new vulnerability is discovered in a monitored dependency
  • POLICY_VIOLATION — Triggered when a scan result violates an organization policy
  • SCAN_COMPLETE — Triggered when a scan finishes, with summary results
  • FIX_AVAILABLE — Triggered when a fix becomes available for a previously unfixable vulnerability

Webhook Handler Example

# Express.js webhook handler
app.post('/mend-webhook', (req, res) => {
  const event = req.body;

  switch (event.type) {
    case 'NEW_VULNERABILITY':
      if (event.severity === 'CRITICAL') {
        // Create PagerDuty incident
        createPagerDutyIncident(event);
        // Create Jira ticket
        createJiraTicket(event);
      }
      break;

    case 'POLICY_VIOLATION':
      // Notify team in Slack
      sendSlackAlert(event);
      break;

    case 'SCAN_COMPLETE':
      // Update dashboard metrics
      updateDashboard(event.summary);
      break;
  }

  res.status(200).send('OK');
});
