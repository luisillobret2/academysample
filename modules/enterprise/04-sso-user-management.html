Enterprise Identity Integration

Enterprise organizations require seamless integration with their identity providers for secure, scalable user management. Mend supports SAML 2.0, OIDC, and SCIM for complete identity lifecycle management.

Supported Identity Providers

👤

Okta

SAML 2.0 and OIDC integration with automated user provisioning via SCIM 2.0.

👤

Azure AD / Entra ID

Enterprise SSO with group-based role mapping and conditional access policies.

👤

Google Workspace

Google SSO with organization unit-based access control.

👤

OneLogin / PingIdentity

Standard SAML/OIDC integration with JIT provisioning support.

SAML SSO Configuration

# Mend SAML SSO Configuration
# Navigate to: Organization Settings > Authentication > SSO

# Required SAML attributes:
# - NameID: user email address
# - firstName: user first name
# - lastName: user last name
# - groups: comma-separated group names (for role mapping)

# Mend SP metadata:
# Entity ID: https://app.mend.io/saml/metadata
# ACS URL: https://app.mend.io/saml/acs
# SLO URL: https://app.mend.io/saml/slo

# Okta configuration example:
# 1. Create new SAML 2.0 application in Okta
# 2. Set Single Sign On URL to ACS URL above
# 3. Set Audience URI to Entity ID above
# 4. Map attributes: email, firstName, lastName, groups
# 5. Assign users/groups to the application

Automated User Provisioning (SCIM)

SCIM (System for Cross-domain Identity Management) automates user lifecycle management:

  • Just-In-Time Provisioning — Users are automatically created in Mend on first SSO login
  • SCIM Provisioning — Proactive user/group sync from your IdP to Mend
  • Deprovisioning — Users removed from IdP groups are automatically deactivated in Mend
  • Group Sync — IdP groups map to Mend roles and permissions automatically

Role-Based Access Control

Mend provides granular RBAC to ensure users have appropriate access levels:

  • Organization Admin — Full control over organization settings, policies, and user management
  • Security Admin — Manage security policies, view all findings, configure integrations
  • Project Admin — Manage specific projects, view findings for assigned projects
  • Developer — View findings for their repositories, resolve/dismiss findings
  • Viewer — Read-only access to dashboards and reports

💡 Security Best Practice

Always enforce SSO for all users and disable password-based login. This ensures that user access is governed by your central identity provider and its policies (MFA, conditional access, session management).
