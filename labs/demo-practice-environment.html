Study the standard Mend demo flow before practicing. A structured demo moves through introduction, SCA scan, vulnerability review, AI remediation, SAST scan, and policy configuration in a predictable, repeatable sequence.

# Clone the demo materials repository git clone https://github.com/mend-io/demo-kit.git ~/demo-kit cd ~/demo-kit # Open the standard demo script cat docs/demo-script.md # Review the demo flow outline cat docs/demo-flow.md
$ cat docs/demo-flow.md Mend Standard Demo Flow 1. Introduction & value proposition (2 min) 2. SCA scan on sample project (3 min) 3. Vulnerability review & prioritization (3 min) 4. AI remediation demonstration (3 min) 5. SAST scan & taint analysis (2 min) 6. Policy configuration (2 min) Total target time: 15 minutes
Tip Memorize the six-phase flow and target timings. A polished demo stays within 15 minutes and smoothly transitions between phases without fumbling for the next screen.

Set Up the Demo Environment

Prepare a clean demo workspace with the sample projects and demo-specific settings. A dedicated environment ensures no leftover scan data confuses your demonstration.

# Create a fresh demo workspace mkdir -p ~/demo-workspace && cd ~/demo-workspace # Copy the sample projects from the demo kit cp -r ~/demo-kit/projects/* . # Configure demo-specific Mend settings mend config set org "Mend Demo Org" mend config set project-prefix "Demo" mend config set ai-remediation enabled # Authenticate with the demo organization mend auth --login --org "Mend Demo Org"
$ mend config set org "Mend Demo Org" Organization set: Mend Demo Org $ cp -r ~/demo-kit/projects/* . Copied 3 sample projects: java-vulnerable-app (SCA demo) python-web-app (SAST demo) monorepo-example (policy demo) $ mend auth --login --org "Mend Demo Org" Authenticated to Mend Demo Org. AI remediation: enabled
Notice Always use a dedicated demo organization so customer data and real findings never appear in your demo. Reset the workspace before each practice run for consistent results.
  • ✓ Created a clean demo workspace with all 3 sample projects
  • ✓ Configured demo-specific Mend settings and AI remediation
  • ✓ Authenticated to the dedicated demo organization
3

Practice the SCA Demo Flow

Run through the SCA portion of the demo: scan the Java application, review vulnerabilities, show AI remediation, and generate an SBOM. Time yourself to stay within the 8-minute SCA budget.

# Start the SCA demo timer time mend scan --product "Demo: Java App" --type sca # Show the vulnerability dashboard (open in browser) mend dashboard --product "Demo: Java App" # Demonstrate AI remediation on a critical vulnerability mend remediate --suggest --severity critical --ai # Generate SBOM for the demo mend sbom export --format cyclonedx --output demo-sbom.json
$ time mend scan --product "Demo: Java App" --type sca Scanning: java-vulnerable-app Found 47 direct dependencies, 183 total Critical: 3 High: 7 Medium: 12 Low: 5 Scan uploaded. Dashboard ready. real 1m12s $ mend remediate --suggest --severity critical --ai CVE-2021-44228 (Log4Shell) - log4j-core 2.14.1 AI Fix: Upgrade to log4j-core 2.17.1 Confidence: 98% | Auto-applicable: yes
Tip Narrate each action as you perform it. Customers want to see the workflow, not just the result. Highlight the AI remediation confidence score and the one-click apply capability as a key differentiator.
  • ✓ Completed the SCA scan within the 8-minute budget
  • ✓ Demonstrated AI remediation with a critical vulnerability
  • ✓ Generated and showed the SBOM export
4

Practice the SAST Demo Flow

Run through the SAST portion: scan the Python web application, review findings, show taint analysis tracing untrusted input to a dangerous sink, and apply an AI-suggested fix. Time yourself to stay within the 4-minute SAST budget.

# Run the SAST scan on the Python web app time mend scan --product "Demo: Python Web App" --type sast # Open the SAST findings dashboard mend dashboard --product "Demo: Python Web App" --view sast # Show taint analysis for a SQL injection finding mend sast trace --finding SQLI-001 # Apply the AI-suggested fix mend remediate --apply --finding SQLI-001 --ai
$ time mend scan --product "Demo: Python Web App" --type sast Scanning: python-web-app (42 files) SAST findings: Critical: 2 (SQL Injection, Command Injection) High: 4 Medium: 8 Scan complete. real 0m48s $ mend sast trace --finding SQLI-001 Taint trace: SQL Injection (SQLI-001) Source: request.args["id"] (app.py:34) -> passes through validate_input() (app.py:40) Sink: cursor.execute(f"SELECT * FROM users WHERE id={id}") (app.py:52) AI Fix: Use parameterized query cursor.execute("SELECT * FROM users WHERE id=?", (id,))
Tip Taint analysis is a strong selling point. Walk the customer through the source-to-sink trace visually, then show the AI fix replacing the vulnerable code with a parameterized query in one click.
  • ✓ Completed the SAST scan within the 4-minute budget
  • ✓ Demonstrated taint analysis tracing source to sink
  • ✓ Applied the AI-suggested fix for the SQL injection finding

$ cat ~/demo-kit/docs/common-questions.md Top Customer Questions & Approved Answers Q: How is Mend priced? A: Per-developer pricing with unlimited scans and repos. Enterprise tiers include SSO, RBAC, and priority support. Q: How does Mend compare to Snyk?
