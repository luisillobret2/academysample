Jenkins Pipeline Integration

Guided
🕓 60 min 🏆 100 XP 🔄 Integration 📚 5 steps

🎯 Lab Objectives

  • Install and configure the Mend Jenkins plugin
  • Write a Jenkinsfile with a Mend scan stage
  • Configure build quality gates and fail conditions
  • Run a pipeline and interpret console output
  • View Mend reports and trends in Jenkins
1

Install Mend Jenkins Plugin

Install the Mend plugin from the Jenkins plugin repository and configure global credentials for the Mend platform.

// Navigate to Jenkins > Manage Jenkins > Plugins > Available plugins // Search for "Mend" and install the Mend plugin // Restart Jenkins after installation
Jenkins > Manage Jenkins > Credentials > System > Global credentials Add Credentials Kind: Secret text ID: MEND_API_KEY Secret: <your Mend API key> Credentials saved successfully.
Tip Store the Mend URL, email, and API key as global credentials so pipelines can reference them without exposing secrets in code.
  • ✓ Mend plugin installed and Jenkins restarted
  • ✓ Global credentials created for Mend API key, email, and URL
2

Create a Jenkinsfile with Mend Scanning

Add a Mend scan stage to your Jenkinsfile. The stage uses the mendScan step to run SCA and SAST scans.

pipeline { agent any environment { MEND_EMAIL = credentials('mend-email') MEND_KEY = credentials('mend-api-key') MEND_URL = credentials('mend-url') } stages { stage('Checkout') { steps { checkout scm } } stage('Build') { steps { sh 'mvn clean package' } } stage('Mend Scan') { steps { mendScan product: 'Lab: Java App', type: 'sca,sast' } } } }
$ git add Jenkinsfile $ git commit -m "Add Mend scan stage" [main 4a2b9c1] Add Mend scan stage
Tip Use the type argument to select one or more scan types. Separate multiple values with commas, for example sca,sast.
  • ✓ Jenkinsfile created in the repository root
  • ✓ Mend scan stage uses the correct product and scan type
3

Configure Build Quality Gates

Add fail conditions and post-scan actions to enforce security standards. Fail the build when critical vulnerabilities are found and publish results.

pipeline { agent any environment { MEND_EMAIL = credentials('mend-email') MEND_KEY = credentials('mend-api-key') MEND_URL = credentials('mend-url') } stages { stage('Build') { steps { sh 'mvn clean package' } } stage('Mend Scan') { steps { mendScan product: 'Lab: Java App', type: 'sca,sast', failOn: 'critical' } } } post { always { mendPublishReports() } failure { echo 'Build failed due to critical security findings.' } } }
[Mend Scan] Running Mend scan... ERROR: Critical vulnerabilities detected. Failing build. Build failed due to critical security findings.
Notice Quality gates help prevent vulnerable code from reaching production. Start with failOn: 'critical' and tighten thresholds as the team matures.
  • ✓ Fail condition set to fail on critical vulnerabilities
  • ✓ Post-scan publishing action added
  • ✓ Pipeline intentionally fails when critical findings exist
4

Run the Pipeline

Trigger a build from the Jenkins job page and watch the console output for scan progress and results.

// In Jenkins, click "Build Now" on the job page // Open the build console output
Started by user learner [Pipeline] stage [Pipeline] { (Mend Scan) Scanning project: Lab: Java App Detected: Maven (pom.xml) Resolving dependencies... Found 47 direct dependencies, 183 total Critical: 3 vulnerabilities High: 7 vulnerabilities Results uploaded to Mend platform. [Pipeline] } [Pipeline] // stage Finished: SUCCESS
Tip If the build fails, expand the Mend Scan stage in the console log to see the exact vulnerability IDs and remediation guidance.
  • ✓ Build triggered successfully
  • ✓ Mend Scan stage completed in console output
  • ✓ Build status reflects the configured quality gate
5

View Reports in Jenkins

After the build finishes, access the Mend scan reports from the Jenkins build page and enable trend tracking over time.

// Build page > Mend Scan Results // View vulnerabilities, severity breakdown, and remediation links
Mend Scan Results Critical: 0 High: 2 Medium: 8 Low: 4 Trend: Critical findings decreased from 3 to 0.
Why trends matter Trend tracking helps teams measure security posture improvements and spot regressions before they reach production.
  • ✓ Opened Mend scan report from the Jenkins build page
  • ✓ Reviewed severity trend across builds

🧠 Knowledge Check

Which Jenkinsfile step runs the Mend scan?
  • mend()
  • mendScan product: 'Lab: Java App', type: 'sca,sast'
  • scanMend()
  • mendPipeline()
What is the purpose of the failOn parameter?
  • To skip the build stage
  • To fail the build when a specified severity threshold is met
  • To disable SAST scanning
  • To publish reports to the Mend platform
