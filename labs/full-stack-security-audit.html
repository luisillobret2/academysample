Full Stack Security Audit

Challenge
🕓 90 min 🏆 200 XP 🔄 SCA + SAST 📚 5 steps

🎯 Lab Objectives

  • Assess a full-stack application and identify its attack surface
  • Run combined SCA and SAST scans across all components
  • Correlate SCA and SAST findings to uncover compound risks
  • Prioritize remediation by risk score, effort, and business impact
  • Present an executive audit report with metrics and recommendations
1

Assess the Application

Review the target full-stack application. It consists of a Node.js backend API, a React frontend, and a Python microservice. Identify languages, dependency managers, and the attack surface.

# Application structure fullstack-app/ backend/ # Node.js + Express, package.json frontend/ # React, package.json microservice/ # Python + Flask, requirements.txt
$ find . -name "package.json" -o -name "requirements.txt" -o -name "*.js" | head -n 20 ./backend/package.json ./frontend/package.json ./microservice/requirements.txt ./backend/src/routes/auth.js ./frontend/src/components/Login.jsx ./microservice/app.py
Tip Map each component to its language, framework, and dependency file before selecting scan types. This ensures every entry point is covered.
  • ✓ Mapped all three application components
  • ✓ Identified dependency files for each component
2

Run Combined SCA + SAST Scans

Run both SCA and SAST scans across the backend, frontend, and microservice. Use the Mend CLI in each component directory.

# Backend scan cd backend mend scan --product "Full Stack Audit" --type sca,sast --name "backend-node" # Frontend scan cd ../frontend mend scan --product "Full Stack Audit" --type sca,sast --name "frontend-react" # Microservice scan cd ../microservice mend scan --product "Full Stack Audit" --type sca,sast --name "microservice-python"
Scanning backend-node Detected: npm (package.json) Critical: 2 vulnerabilities SAST findings: 4 high, 6 medium Scanning frontend-react High: 3 vulnerabilities SAST findings: 2 high, 8 medium Scanning microservice-python Critical: 1 vulnerability SAST findings: 1 high, 3 medium
Notice Running both SCA and SAST together reveals two different risk classes: vulnerable dependencies and risky code patterns. Both are needed for a complete audit.
  • ✓ SCA and SAST scans completed for all three components
  • ✓ All scan results uploaded to the Mend platform under the same product
  • ✓ Recorded vulnerability and SAST finding counts by component
3

Correlate Findings

Cross-reference SCA and SAST findings to identify compound risks. A vulnerable dependency combined with code that uses it unsafely is much more dangerous than either finding alone.

# Correlation table | Component | SCA Finding | SAST Finding | Compound Risk | |-----------|-------------|--------------|---------------| | Backend | CVE-2023-45857 (axios) | SSRF in request helper | Attacker can pivot SSRF through vulnerable axios version | | Frontend | CVE-2023-2249 (React) | XSS in user input rendering | XSS payload may bypass patched React if input is unsanitized | | Microservice | CVE-2023-20871 (Flask) | Unsafe deserialization | Remote code execution via vulnerable Flask + pickle usage |
Compound risks identified: 3 Backend: CVE-2023-45857 + SSRF = High risk Frontend: CVE-2023-2249 + XSS = High risk Microservice: CVE-2023-20871 + pickle = Critical risk
Tip When SCA shows a vulnerable library and SAST shows the same library being used in a dangerous way, prioritize remediation immediately.
  • ✓ Created a correlation table for all components
  • ✓ Identified at least one compound risk per component
4

Prioritize and Plan Remediation

Build a remediation matrix that sorts findings by risk score, remediation effort, and business impact. Start with the highest-risk, lowest-effort items.

# Remediation matrix | Finding | Severity | Effort | Business Impact | Risk Score | Priority | |---------|----------|--------|-----------------|------------|----------| | Microservice RCE | Critical | Low | High | 10 | P1 | | Backend SSRF | High | Medium | High | 8 | P2 | | Frontend XSS | High | Low | Medium | 7 | P3 | | Backend dep upgrade | Medium | Low | Low | 4 | P4 |
P1: Patch Flask and remove unsafe pickle in microservice P2: Upgrade axios and sanitize SSRF-prone URL builder in backend P3: Upgrade React and sanitize user input in frontend P4: Remaining dependency upgrades and SAST clean-up
Tip Use a risk score that combines severity, exploitability, and business impact. This prevents teams from being overwhelmed by low-value findings.
  • ✓ Remediation matrix created with priorities
  • ✓ P1 items are compound risks with low effort and high impact
  • ✓ Remediation plan reviewed with a teammate or stakeholder
5

Present the Audit Report

Create an executive summary that communicates the audit scope, key metrics, risk heatmap, and prioritized recommendations.

# Executive summary structure # Full Stack Security Audit Report - Scope: Node.js backend, React frontend, Python microservice - Scan types: SCA + SAST across all components - Key metrics: - Critical findings: 3 - High findings: 7 - Compound risks: 3 - Remediation rate target: 80% within 30 days - Risk heatmap: Microservice (Critical), Backend (High), Frontend (High) - Recommendations: Patch P1 items immediately, schedule P2 within 2 weeks, track progress weekly
Executive Summary generated Audience: CTO, Engineering Leads, Security Team Report exported to fullstack_audit_report.pdf
Tip Executive reports should lead with business impact and end with clear actions. Avoid jargon; translate risk into potential operational and financial impact.
  • ✓ Executive summary includes scope, metrics, and recommendations
  • ✓ Risk heatmap clearly highlights the most critical component
  • ✓ Report is ready to share with leadership

🧠 Knowledge Check

What makes a compound risk especially dangerous?
  • It is only detected by SCA
  • A vulnerable dependency is combined with unsafe code that uses it
  • It has no CVSS score
  • It only affects frontend applications
Which factor should most influence remediation priority?
  • Number of lines of code changed
  • Risk score combining severity, exploitability, and business impact
  • The date the vulnerability was published
  • The programming language of the finding

🎉 Lab Complete!

You have completed a full-stack security audit, correlated SCA and SAST findings, and presented an executive remediation report.
