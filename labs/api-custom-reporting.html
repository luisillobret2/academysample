Get Your API Credentials

Generate an API key in the Mend platform and authenticate REST API requests using a Bearer token in the Authorization header.

# Mend platform > Settings > API Keys > Generate Key # Copy the user key and export it as an environment variable export MEND_API_KEY="your-api-key" export MEND_URL="https://app.mend.io"
$ curl -H "Authorization: Bearer ${MEND_API_KEY}" \ "https://app.mend.io/api/v3/projects" HTTP/1.1 200 OK Content-Type: application/json { "total": 12, "projects": [...] }
Security note Treat API keys as secrets. Never commit them to source control; store them in environment variables or a secrets manager.
  • ✓ Mend API key generated and copied
  • ✓ Authenticated API request returned HTTP 200

Explore the Mend REST API

Use curl to list projects, retrieve scan results, and fetch vulnerability details. Study the JSON response structure before writing scripts.

# List projects curl -H "Authorization: Bearer ${MEND_API_KEY}" \ "https://app.mend.io/api/v3/projects" # Get vulnerabilities for a specific project curl -H "Authorization: Bearer ${MEND_API_KEY}" \ "https://app.mend.io/api/v3/projects/{projectId}/vulnerabilities" # Get a specific vulnerability detail curl -H "Authorization: Bearer ${MEND_API_KEY}" \ "https://app.mend.io/api/v3/vulnerabilities/{vulnId}"
{ "projects": [ { "id": "proj-123", "name": "java-vulnerable-app", "product": "Lab: Java App", "lastScan": "2025-06-15T10:00:00Z" } ] }
Tip Use tools like jq or Postman to format JSON responses and explore the schema interactively.

  • ✓ Retrieved vulnerability data for a project
  • ✓ Identified the JSON fields needed for reporting

Write a Custom Report Script

Create a Python script that pulls vulnerability data from the Mend API and writes it to a CSV file for stakeholders.

import csv import os import requests MEND_URL = os.environ.get('MEND_URL', 'https://app.mend.io') API_KEY = os.environ['MEND_API_KEY'] HEADERS = {'Authorization': f'Bearer {API_KEY}'} def get_vulnerabilities(project_id): url = f'{MEND_URL}/api/v3/projects/{project_id}/vulnerabilities' resp = requests.get(url, headers=HEADERS) resp.raise_for_status() return resp.json() def export_csv(project_id, filename): vulns = get_vulnerabilities(project_id) with open(filename, 'w', newline='') as f: writer = csv.writer(f) writer.writerow(['CVE', 'Severity', 'Component', 'Fixed Version']) for v in vulns: writer.writerow([ v.get('cveId'), v.get('severity'), v.get('componentName'), v.get('fixVersion') ]) export_csv('proj-123', 'vulnerabilities.csv')
$ python export_vulnerabilities.py Fetched 42 vulnerabilities. Saved report to vulnerabilities.csv
Tip Add error handling, pagination, and retries for production scripts. Use environment variables for all secrets.
  • ✓ Python script created and executable
  • ✓ CSV report generated from API data

Build a Dashboard Query

Aggregate vulnerability metrics across projects to create an executive summary dashboard. Use matplotlib or a charting library to visualize the data.

import requests import os import matplotlib.pyplot as plt MEND_URL = os.environ.get('MEND_URL', 'https://app.mend.io') API_KEY = os.environ['MEND_API_KEY'] HEADERS = {'Authorization': f'Bearer {API_KEY}'} def get_projects(): resp = requests.get(f'{MEND_URL}/api/v3/projects', headers=HEADERS) resp.raise_for_status() return resp.json()['projects'] def summarize(): total = critical = fixed = 0 for project in get_projects(): vulns = requests.get( f"{MEND_URL}/api/v3/projects/{project['id']}/vulnerabilities", headers=HEADERS ).json() total += len(vulns) critical += sum(1 for v in vulns if v['severity'] == 'critical') fixed += sum(1 for v in vulns if v['status'] == 'resolved') remediation_rate = (fixed / total * 100) if total else 0 print(f'Total: {total}, Critical: {critical}, Remediation Rate: {remediation_rate:.1f}%') return total, critical, remediation_rate summary = summarize() plt.bar(['Total', 'Critical', 'Remediation Rate'], summary) plt.savefig('executive_dashboard.png')
$ python dashboard.py Total: 142, Critical: 5, Remediation Rate: 67.6% Dashboard saved to executive_dashboard.png
Tip Executive dashboards are most effective when they track a small set of meaningful KPIs: critical count, remediation rate, and mean time to remediate.
  • ✓ Aggregated metrics across all projects
  • ✓ Visualized total vulnerabilities, critical count, and remediation rate

Automate Weekly Reports

Schedule the report script to run weekly and email the results. Use cron on Linux/macOS or Task Scheduler on Windows.

# weekly_report.sh #!/bin/bash export MEND_API_KEY="$MEND_API_KEY" python3 /opt/mend-reports/dashboard.py > /opt/mend-reports/weekly.log mail -s "Weekly Mend Security Report" -A /opt/mend-reports/executive_dashboard.png \ security-team@company.com < /opt/mend-reports/weekly.log
# Crontab entry: every Monday at 09:00 0 9 * * 1 /opt/mend-reports/weekly_report.sh
Tip Store the email configuration in an environment file or a secrets manager. Verify the script runs manually before enabling the schedule.
  • ✓ Shell wrapper created to run the report and send email
  • ✓ Cron job scheduled for weekly execution

Export data in JSON or CSV format so BI tools such as Tableau, Power BI, or Looker can consume it for further analysis.

# Export projects summary as JSON curl -H "Authorization: Bearer ${MEND_API_KEY}" \ "https://app.mend.io/api/v3/projects" \ -o projects.json # Export vulnerabilities as CSV using the Python script python3 export_vulnerabilities.py

Tip BI tools can connect directly to CSV or JSON files, or to a database that you populate with a scheduled ETL job. Pick the option that fits your analytics stack.
