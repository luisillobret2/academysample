SCA Policy Configuration

Guided
🕓 45 min 🏆 100 XP 🔄 SCA 📚 5 steps

🎯 Lab Objectives

  • Explore the Mend policy engine and policy types
  • Create a security policy that blocks critical vulnerabilities
  • Create a license policy that flags copyleft licenses
  • Assign policies to a project
  • Test policy enforcement through a live scan
1

Explore the Policy Engine

Open the Mend platform and navigate to the Policy tab. Here you can define rules that govern what is acceptable in your projects. Mend distinguishes between two policy families:

  • Security policies act on vulnerability severity, CVSS score, or exploit maturity.
  • License policies act on dependency licenses, such as copyleft or proprietary restrictions.

To see the policies already available in your organization, use the Mend CLI:

# List all policies in the organization mend policy list
$ mend policy list Organization: Mend Academy Demo Total policies: 4 ID Name Type Scope Action -- ---- ---- ----- ------ pol-1 Default Security security org warn pol-2 Default License license org warn pol-3 Block Critical security project block pol-4 Flag Copyleft license project warn
Tip Policies can be scoped to an organization or to specific projects. Start with project-scoped policies to test changes before rolling them out across the organization.
  • ✓ Navigated to the Policy tab in the Mend platform
  • ✓ Listed existing policies with mend policy list
  • ✓ Identified the difference between security and license policies
2

Create a Security Policy

Create a security policy that blocks any dependency with a critical vulnerability. This prevents vulnerable code from merging into protected branches.

# Create a security policy that blocks critical findings mend policy create --name "Block Critical" --type security --severity critical --action block
$ mend policy create --name "Block Critical" --type security --severity critical --action block Creating policy... Policy created successfully. ID: pol-5 Name: Block Critical Type: security Severity threshold: critical Action: block
Important The block action fails the build and prevents merging in most CI/CD integrations. Make sure the policy scope is limited to the target project until the team is ready to enforce it more broadly.
  • ✓ Created the "Block Critical" security policy
  • ✓ Set the action to block and severity to critical
3

Create a License Policy

License policies help legal and compliance teams identify dependencies that may introduce intellectual-property risks. Create a policy that flags the GPL and LGPL copyleft licenses.

# Create a license policy that flags copyleft licenses mend policy create --name "Flag Copyleft" --type license --licenses GPL,LGPL --action warn
$ mend policy create --name "Flag Copyleft" --type license --licenses GPL,LGPL --action warn Creating policy... Policy created successfully. ID: pol-6 Name: Flag Copyleft Type: license Licenses: GPL, LGPL Action: warn
Tip The warn action alerts the team but does not block the build. Use it for license compliance while stakeholders review and approve exceptions.
  • ✓ Created the "Flag Copyleft" license policy
  • ✓ Added GPL and LGPL to the license list
  • ✓ Chose the warn action
4

Apply Policies to a Project

Policies only take effect when they are assigned to a project. Assign the two policies you created to the sample Java application project.

# Assign policies to the sample project mend policy assign --project "Lab: Java App" --policies "Block Critical,Flag Copyleft"
$ mend policy assign --project "Lab: Java App" --policies "Block Critical,Flag Copyleft" Resolving project: Lab: Java App Policies to assign: Block Critical, Flag Copyleft 2 policies assigned successfully. Project scope updated.
Tip You can view the policies assigned to a project from the project settings page or by running mend policy list --project "Lab: Java App".
  • ✓ Assigned both policies to the "Lab: Java App" project
  • ✓ Verified the assignment in the project settings
5

Test Policy Enforcement

Run a fresh SCA scan on the project. The new policies will evaluate the scan results, and any violation will be shown in the terminal and the platform dashboard.

# Run an SCA scan to evaluate policy enforcement mend scan --project "Lab: Java App" --type sca
$ mend scan --project "Lab: Java App" --type sca Scanning project: Lab: Java App Detected: Maven (pom.xml) Resolving dependencies... Found 47 direct dependencies, 183 total Policy evaluation: [Block Critical] log4j-core 2.14.1: critical CVE-2021-44228 [Block Critical] snakeyaml 1.29: critical CVE-2022-25857 [Flag Copyleft] org.apache.commons:commons-lang3: LGPL Policy violations: 3 Build action: block Results uploaded to Mend platform. View at: https://app.mend.io/projects/lab-java-app
Notice The two critical vulnerabilities triggered the Block Critical policy, causing the build to be blocked. The license policy triggered a warning for the LGPL dependency. Review these findings in the dashboard before fixing the policy or the dependency.
  • ✓ Ran the SCA scan on the project
  • ✓ Observed the security policy violations
  • ✓ Observed the license policy warning

Which action prevents a build from passing when a policy is violated?
