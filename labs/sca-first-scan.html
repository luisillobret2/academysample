Your First SCA Scan

Guided
🕓 45 min 🏆 100 XP 🔄 SCA 📚 6 steps

🎯 Lab Objectives

  • Install and configure the Mend CLI
  • Create a project and connect a repository
  • Run your first SCA scan on a Java application
  • Interpret vulnerability results and severity ratings
  • Apply remediation recommendations
  • Generate an SBOM export
1

Install the Mend CLI

The Mend CLI is the primary tool for running scans locally and in CI/CD pipelines. Start by installing it on your machine.

# Install Mend CLI via npm npm install -g @mend/cli # Verify installation mend --version
$ mend --version Mend CLI v3.8.2 Copyright (c) Mend.io. All rights reserved.
Tip If you get a permission error on macOS/Linux, try using sudo npm install -g @mend/cli or use a node version manager like nvm.
  • ✓ Mend CLI installed successfully
  • ✓ mend --version returns a version number
2

Authenticate with Your Mend Account

Before scanning, authenticate with your Mend platform credentials. This links your scans to your organization and projects.

# Authenticate with Mend platform mend auth --login # Or use environment variables for CI/CD export MEND_EMAIL="your@email.com" export MEND_KEY="your-api-key" export MEND_URL="https://app.mend.io"
$ mend auth --login Opening browser for authentication... Authentication successful! Organization: Mend Academy Demo User: learner@mend.io
  • ✓ Authenticated with Mend platform
  • ✓ Organization name appears in output
3

Create a Project and Run the Scan

Navigate to the sample Java application and run your first SCA scan. The CLI will detect the pom.xml and scan all dependencies.

# Navigate to the sample project cd ~/labs/java-vulnerable-app # Run SCA scan mend scan --product "Lab: Java App" --type sca
$ mend scan --product "Lab: Java App" --type sca Scanning project: java-vulnerable-app Detected: Maven (pom.xml) Resolving dependencies... Found 47 direct dependencies, 183 total Scan complete. Results: Critical: 3 vulnerabilities High: 7 vulnerabilities Medium: 12 vulnerabilities Low: 5 vulnerabilities Policy violations: 2 License issues: 1 Results uploaded to Mend platform. View at: https://app.mend.io/products/java-vulnerable-app
Notice The scan found 3 critical vulnerabilities. In a real environment, these would need immediate attention. We'll address them in the next steps.
  • ✓ Scan completed without errors
  • ✓ Results uploaded to the Mend platform
  • ✓ Noted the vulnerability counts by severity
4

Review Vulnerability Results

Open the Mend platform dashboard to review the scan results. Click on each critical vulnerability to see details, CVSS score, and remediation guidance.

In the dashboard, you'll see:

  • Vulnerability list sorted by severity with CVSS scores
  • Dependency tree showing which component introduces each vulnerability
  • Remediation path with recommended fixed versions
  • Policy violations for any violated security rules
Key terms CVSS Score: Common Vulnerability Scoring System (0-10). Critical = 9.0+, High = 7.0-8.9, Medium = 4.0-6.9, Low = 0.1-3.9.
  • ✓ Opened the vulnerability dashboard
  • ✓ Reviewed all 3 critical vulnerabilities
  • ✓ Identified the root dependency for each critical finding
5

Apply Remediation

Mend provides automated remediation recommendations. Let's apply the suggested fixes for the critical vulnerabilities.

# View remediation suggestions mend remediate --suggest --severity critical # Apply remediation (updates pom.xml) mend remediate --apply --severity critical
$ mend remediate --suggest --severity critical Critical vulnerability: CVE-2021-44228 (Log4Shell) Component: log4j-core 2.14.1 Fix: Upgrade to log4j-core 2.17.1 Critical vulnerability: CVE-2022-23405 Component: snakeyaml 1.29 Fix: Upgrade to snakeyaml 1.32 Critical vulnerability: CVE-2021-37136 Component: netty-codec-http 4.1.63.Final Fix: Upgrade to netty-codec-http 4.1.86.Final $ mend remediate --apply --severity critical Updated pom.xml with 3 dependency upgrades. Run 'mvn clean install' to verify the build.
  • ✓ Reviewed remediation suggestions
  • ✓ Applied all 3 critical fixes
  • ✓ Verified pom.xml was updated
6

Generate an SBOM Export

Software Bills of Materials (SBOMs) are essential for compliance and transparency. Generate one from your scan results.

# Generate SBOM in CycloneDX format mend sbom export --format cyclonedx --output sbom.json # Generate SBOM in SPDX format mend sbom export --format spdx --output sbom.spdx
$ mend sbom export --format cyclonedx --output sbom.json Generating SBOM... Format: CycloneDX 1.4 Components: 183 Vulnerabilities: 27 SBOM written to sbom.json (42.3 KB)
Why SBOMs matter Many compliance frameworks (Executive Order 14028, EU CRA) require SBOMs. Mend supports both CycloneDX and SPDX formats.
  • ✓ Generated CycloneDX SBOM
  • ✓ Verified the SBOM contains all 183 components

🧠 Knowledge Check

What CVSS score range corresponds to "Critical" severity?
  • 7.0 - 8.9
  • 9.0 - 10.0
  • 4.0 - 6.9
  • 0.1 - 3.9
Which command generates an SBOM in SPDX format?
  • mend scan --format spdx
  • mend sbom export --format spdx --output sbom.spdx
  • mend export --spdx
  • mend report --format spdx

You've successfully run your first SCA scan, reviewed vulnerabilities, applied remediation, and generated an SBOM.
