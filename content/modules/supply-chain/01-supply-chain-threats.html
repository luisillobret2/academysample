The Software Supply Chain

Modern applications are built on a deep supply chain of open-source components. The average enterprise application contains over 80% open-source code, with hundreds of direct and transitive dependencies. This creates a massive attack surface that threat actors increasingly exploit.

Key Supply Chain Attack Vectors

📦

Dependency Confusion

Attackers publish malicious packages to public registries using internal package names, exploiting package manager resolution order.

⌨️

Typosquatting

Malicious packages with names similar to popular libraries (e.g., lod-ash vs lodash) to trick developers.

🔒

Compromised Maintainers

Attackers gain control of legitimate maintainer accounts to inject malicious code into trusted packages.

⚠️

Malicious Updates

A legitimate package is updated with malicious code — as seen in the event-stream and ua-parser-js incidents.

Dependency Risk Scoring

Mend evaluates the risk of each dependency based on multiple factors beyond just known vulnerabilities:

  • Vulnerability History — How many CVEs have been found in this package, and how quickly were they fixed?
  • Maintainer Health — Is the project actively maintained? How many contributors does it have?
  • License Risk — Does the license impose restrictions that conflict with your organization's policies?
  • Popularity & Usage — Widely-used packages get more scrutiny, but abandoned popular packages are high risk
  • Provenance & Integrity — Does the package have verifiable build provenance? Are releases signed?
